# Contributing

这个仓库维护的是本地 AI Agent 工具链自动化流程，重点不是收集某个用户机器上的私有配置，而是沉淀可复用、可验证、可脱敏的操作方法。

## 维护者工作流

1. 先确认问题是否属于本仓库范围：
   - 多客户端 Skill 同步。
   - Claude、Codex、Gemini、OpenCode 等本地 Agent 客户端的共享工具链配置。
   - MCP 接入、传输类型判断、initialize/tools-list 验证。
   - macOS GUI 应用与终端 PATH 不一致导致的本地工具不可用问题。
2. 在本地复现或验证：
   - 查看真实路径、符号链接和配置文件。
   - 运行相关脚本的 `status` 或 `--help`。
   - 对 MCP 工具使用协议级初始化或工具列表检查，而不是只看进程是否启动。
3. 做最小改动：
   - 只修改和问题直接相关的脚本、模板或文档。
   - 不提交 token、cookie、OAuth 文件、私有数据库、完整聊天记录或真实账号隐私。
   - 如果示例需要路径或账号信息，使用脱敏路径和占位符。
4. 验证改动：
   - 脚本改动至少运行 `bash -n scripts/<script-name>`。
   - 文档改动检查链接、命令和文件名是否仍然存在。
   - 示例改动确认不会泄露本机隐私。
5. 更新证据：
   - 如果新增了一个可复用场景，在 `examples/` 下补一个脱敏示例。
   - 如果新增了申请或展示材料，在 `docs/evidence-guide.zh-CN.md` 中补充截图位置或验证方式。

## Pull Request 建议

- 标题说明场景和改动，例如 `Add MCP stdio verification example`。
- 描述里写清楚：
  - 解决的具体问题。
  - 改了哪些文件。
  - 如何验证。
  - 是否包含任何需要用户本地替换的路径。

## 安全边界

不要提交以下内容：

- API key、OAuth token、cookie、SSH 私钥。
- 真实聊天数据库、完整日志、浏览器 profile。
- 带有个人邮箱、手机号、身份证件、付款信息的截图。
- 未经脱敏的企业内部路径、账号和项目名。
